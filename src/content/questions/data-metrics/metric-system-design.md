# 从零搭建产品指标体系

## 面试场景

> 面试官问你：「你接手了一个新产品，目前数据看板上只有DAU和收入两个指标。老板让你在一周内搭建一套完整的指标体系。你会怎么做？」

这道题考察的不是你能背多少指标名词，而是你是否有能力**从业务目标出发，系统性地构建一套分层的、可执行的指标框架**。很多PM能说出AARRR、北极星指标这些概念，但面对一个具体产品时，不知道怎么从零设计。

---

## 核心概念解析

### 什么是指标体系

指标体系不是"一堆指标的集合"，而是**一套有逻辑关系的指标树**：

- **自上而下**：从商业目标到用户行为，层层拆解
- **互相印证**：多个指标交叉看，才能发现真问题
- **可行动**：每个指标的变化都能对应到具体的行动项

> 一个好指标体系的核心价值：当你看到一个指标异常时，能在3步之内定位到问题出在哪。

### 指标体系的三个层次

| 层次 | 含义 | 举例 | 谁看 |
|------|------|------|------|
| 北极星指标 | 全公司对齐的唯一核心指标 | 月活跃付费用户数 | CEO/VP |
| 核心指标 | 衡量关键业务模块的健康度 | 新增、留存、转化、收入 | 产品总监 |
| 过程指标 | 监控具体功能和链路的运行状态 | 注册漏斗各步骤转化率 | PM/运营 |

---

## 三大经典模型

### 模型一：OSM 模型（目标-策略-度量）

**O (Objective) → S (Strategy) → M (Measure)**

从业务目标出发，制定策略，再定义度量指标。

```
O（目标）：提升用户留存
  └─ S（策略1）：优化新用户引导流程
       └─ M（度量）：
          ├─ 引导完成率
          ├─ 引导后首日活跃率
          └─ 7日留存率
  └─ S（策略2）：增加用户互动
       └─ M（度量）：
          ├─ 人均互动次数
          ├─ 互动功能渗透率
          └─ 互动后留存提升
```

**适用场景**：目标明确，需要向下拆解到可度量层级。

### 模型二：UJM 模型（用户旅程地图）

按用户生命周期构建指标：

```
认知 → 兴趣 → 考虑 → 注册 → 激活 → 使用 → 付费 → 留存 → 推荐
 ↓     ↓     ↓     ↓     ↓     ↓     ↓     ↓     ↓
曝光   点击   落地   注册   完成   DAU   转化   续费   NPS
量    率    页停留  率    关键   /WAU  率    率
                 动作
```

**适用场景**：用户链路清晰的产品（电商、SaaS、工具类）。

### 模型三：AARRR + 分层

在经典海盗指标基础上加入分层：

| 阶段 | 指标 | 质量指标 | 数量指标 |
|------|------|---------|---------|
| 获取 | 流量来源 | 优质渠道占比 | 日新增用户数 |
| 激活 | 注册-激活转化 | 激活耗时 | 激活用户数 |
| 留存 | 留存率 | 次日/7日/30日留存 | 周活/月活 |
| 变现 | ARPU | 付费转化率 | 总收入 |
| 推荐 | NPS | 推荐率 | 分享次数 |

**适用场景**：增长阶段的产品，需要找到增长杠杆。

## 从零搭建的六步法

### Step 1：明确产品阶段和商业目标

不同阶段的核心目标完全不同：

| 产品阶段 | 核心目标 | 典型北极星指标 |
|---------|---------|--------------|
| 冷启动期 | 验证PMF | 核心功能周活跃用户 |
| 成长期 | 快速获客 | 月新增活跃用户 |
| 成熟期 | 提效增利 | 月活跃付费用户 |
| 衰退期 | 挽留变现 | 核心用户留存率 |

**面试加分点**：先问"这个产品现在在什么阶段"——不同阶段指标体系完全不同。

### Step 2：定义北极星指标

北极星指标的选取标准（三个"一个"）：
- **一个**指标（不是一组）
- **一个**能代表产品核心价值的指标
- **一个**全公司都能理解和行动的指标

测试方法：如果这个指标涨了10%，公司整体变好了吗？如果不确定，这个指标不够"北极星"。

| 产品类型 | 合适的北极星指标 | 不合适的 |
|---------|----------------|---------|
| 内容平台 | 周均内容消费时长 | DAU（太宽泛） |
| SaaS工具 | 周活跃团队数 | 注册数（只看量不看质） |
| 电商平台 | 月均下单用户数 | GMV（容易被客单价扭曲） |
| 社交产品 | 日均互动用户数 | 注册数（不代表活跃） |

### Step 3：拆解到核心指标（最多5-7个）

以"SaaS工具——周活跃团队数"为例：

```
北极星：周活跃团队数
  ├─ 拉新：新团队注册数
  ├─ 激活：注册到激活的转化率
  ├─ 留存：周→周留存率
  ├─ 扩展：团队内人均使用频次
  └─ 流失：周流失团队数
```

每个核心指标需要定义清楚：
- 口径（怎么算的）
- 周期（日/周/月）
- 对标基线（行业参考值是什么）

### Step 4：设计过程指标和监控指标

过程指标服务于核心指标的诊断：

```
核心指标：周→周留存率下降
  ├─ 过程指标1：功能使用分布（哪个功能使用减少最多）
  ├─ 过程指标2：新功能使用率（是不是新功能体验差导致流失）
  ├─ 过程指标3：竞品替代率（用户是否转向竞品）
  └─ 过程指标4：客诉量变化（是否有集中暴雷的功能）
```

### Step 5：建立指标间的因果关系

**指标不是孤立的，要画出影响关系图**：

```
引导优化 → 激活率↑ → 次日留存↑ → 7日留存↑ → 付费转化↑ → ARPU↑
                                                      ↓
                                                   北极星↑
```

这样当北极星指标变化时，你可以顺着关系图快速定位问题。

### Step 6：设定基线和预警

- **基线**：每个指标的当前值和行业benchmark
- **预警线**：波动超过多少需要关注（通常±10%需要排查）
- **目标线**：季度OKR设定的目标值

## 实战案例：在线教育App指标体系

```
北极星：周完课用户数（每周至少完成1节课的用户数）

核心指标：
├─ 获客：日新增注册（按渠道拆分）
├─ 激活：注册→首次完课转化率
├─ 留存：次周复学率
├─ 变现：试听→付费转化率
└─ 满意：课程评分均值

过程指标（示例）：
├─ 搜索→点击率（内容发现效率）
├─ 视频完播率（内容吸引力）
├─ 作业提交率（参与深度）
├─ 分享课程次数（口碑传播）
└─ 客诉响应时长（服务体验）
```

**数据驱动决策示例**：发现"注册→首次完课转化率"只有15%，拆解发现是"找不到感兴趣的课程"导致的，优化推荐算法后提升到28%。

## 面试常见追问

### 追问1：指标冲突时怎么办？

**场景**：留存率上升但付费转化率下降。

**回答**：
1. 先看时间维度——是否是短期波动
2. 再看用户结构——是不是新来了一批高留存低付费意愿的用户（比如学生群体）
3. 看因果——是否做了某个策略同时影响两个指标
4. 回归北极星——如果北极星在涨，说明指标冲突可能是结构性变化，需要调整指标定义

### 追问2：数据不够怎么办？新冷启动产品没历史数据。

**回答**：
1. 用行业benchmark做基线（竞品数据、行业报告）
2. 先跑一个月数据再定基线（不要用第一天数据做benchmark）
3. 从定性反馈开始——前100个用户的访谈比数据更有价值
4. 随着数据积累逐步完善指标体系，不要一开始就追求完美

### 追问3：如何避免"虚荣指标"？

**虚荣指标的特征**：只升不降、和业务结果无关、让团队自我感觉良好但驱动不了行动。

| 虚荣指标 | 可行动指标 |
|---------|----------|
| 累计注册用户数 | 月活跃用户数 |
| 总下载量 | 7日留存率 |
| 页面总PV | 人均有效停留时长 |
| 社交媒体粉丝数 | 互动率/转化率 |

**检验方法**：看到这个指标的变化，你知道下一步该做什么吗？如果不知道，就是虚荣指标。

## 标准回答框架

面试时遇到"搭建指标体系"类问题：

```
1. 先问背景：产品类型、阶段、商业模式
2. 确定北极星：从商业目标出发，选定一个核心指标
3. 拆解核心指标：用OSM或UJM模型，拆出5-7个核心指标
4. 设计过程指标：每个核心指标配2-3个诊断性过程指标
5. 建立关系：画出指标间的影响关系图
6. 设定基线：给出行业参考值和目标线
7. 预留灵活性：说明指标体系需要随产品阶段迭代
```

---

## 关键收获

1. **指标体系的核心是逻辑关系，不是指标数量**——30个互不相干的指标不如7个有因果关系的指标
2. **北极星指标是锚**——所有指标的设立都要能回答"这如何影响北极星"
3. **先搭骨架再填充**——从目标到策略到度量，自上而下构建，不要从下往上堆
4. **过程指标的诊断能力比核心指标更重要**——能告诉你"问题出在哪"的指标才是好指标
5. **指标体系需要随产品进化**——没有一劳永逸的方案，每季度至少review一次指标体系的适用性
