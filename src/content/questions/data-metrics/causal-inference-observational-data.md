## 面试场景

> 面试官：你负责一款电商 App 的会员功能。上线会员功能后，开通会员的用户后续 30 天消费额比未开通用户高出 65%。市场部认为"会员功能驱动了消费提升"，要求加大推广投入。你怎么评估这个结论是否成立？如果不能做 A/B 测试，你怎么判断真实的因果效果？

---

## 核心陷阱：相关 ≠ 因果

这是数据驱动决策中最经典、也最致命的陷阱。

### 为什么 65% 的差异可能不是因果效果？

**选择性偏差（Selection Bias）：**

愿意开通会员的用户本身就是高消费用户。他们不是因为开了会员才消费更多，而是因为消费多才开了会员。

```
真实因果效果 = 观察差异 - 选择性偏差
65% = ? (因果效果) + ? (选择性偏差)
```

如果不消除选择性偏差，你可能在向一个"无效功能"投入大量资源。

## 回答框架：观察数据的因果推断工具箱

### 方法一：倾向得分匹配（PSM）

**核心思想：** 在两组用户中找到"可比的人"进行比较。

**操作步骤：**

1. **选择协变量**：上线前的消费频次、客单价、品类偏好、活跃天数等
2. **估算倾向得分**：用逻辑回归预测每个用户"开通会员的概率"
3. **匹配**：将开通会员的用户与概率相近的未开通用户配对
4. **比较**：计算匹配后两组的消费差异

**结果解读示例：**
```
原始差异：65% → PSM 后差异：18%
结论：会员功能的真实因果效果约为 18%，
     其余 47% 来自高消费用户的自选择偏差。
```

**局限性：** PSM 只能控制"可观测的"混杂因素，不能处理不可观测的差异。

### 方法二：双重差分法（DID, Difference-in-Differences）

**核心思想：** 利用时间维度的对比消除固定偏差。

**应用场景：** 会员功能在不同城市分批上线（天然的自然实验）。

```
 DID = (会员用户上线后消费 - 会员用户上线前消费)
     - (非会员用户上线后消费 - 非会员用户上线前消费)
```

**关键假设：** 平行趋势假设——如果没有会员功能，两组用户的变化趋势应该是平行的。

**验证方法：** 检查上线前的趋势是否平行。如果不平行，DID 的结论不可靠。

### 方法三：工具变量法（IV）

**核心思想：** 找到一个"只影响是否开通会员、但不直接影响消费"的变量。

**可能的工具变量：**
- 用户注册时的推荐人是否是会员（影响开通概率，但不直接影响消费）
- 某次系统推送的随机时间差（影响看到会员入口的时机）

**条件：**
- **相关性**：工具变量确实影响"是否开通会员"
- **外生性**：工具变量不通过其他路径影响消费

### 方法四：断点回归设计（RDD）

**核心思想：** 如果会员资格有明确的"门槛"（如消费满 500 元自动获得），比较门槛附近的用户。

```
门槛上方（501-550元） vs 门槛下方（450-499元）
→ 这两组用户在消费行为上"几乎一样"
→ 唯一区别是：一组有会员，一组没有
→ 差异即为因果效果
```

## 完整回答模板（面试 6 分钟版）

1. **识别问题**："65% 的差异是观察性差异，不能直接归因为因果效果。最大的混淆因素是选择性偏差——高消费用户更倾向于开通会员。"

2. **提出方案**："理想情况下，我会优先建议做 A/B 测试——在功能上线前随机控制开通权限。但如果不具备条件，我会用以下方法逼近因果推断："

3. **方法选择**：
   - 有上线前数据 → PSM + DID 组合
   - 有明确的自然实验 → 断点回归或 DID
   - 能找到工具变量 → IV 方法

4. **落地实施**：
   - "我会先做 PSM，控制上线前的消费水平、频次、品类等变量，得到匹配后的差异。然后用 DID 验证时间趋势。"
   - "如果 PSM 后差异从 65% 降到 15-20%，我会告诉市场部：会员功能确实有效果，但效果是 15-20%，不是 65%。"

5. **决策建议**："基于 15-20% 的真实效果，重新计算 ROI——如果仍然为正，继续推广但调整预期；如果不够回本，优化会员权益设计后再推广。"

## 实战案例

### 案例：某外卖平台的"超时赔付"功能

**背景：** 上线超时赔付功能后，开通商家的订单增长率比未开通商家高 12%。

**朴素结论：** 超时赔付促进了订单增长。

**因果推断过程：**
1. PSM 匹配后差异从 12% 降到 3%
2. DID 分析显示平行趋势假设成立
3. 最终结论：真实因果效果约 3%，有统计显著性

**决策：** 3% 的提升仍然为正，且赔付功能提升了用户信任，决定保留功能，但不作为主要的增长驱动力推广。

## 面试常见追问

**Q1：A/B 测试不是更简单吗？为什么不直接做？**

现实中很多场景无法做 A/B 测试：(1) 法律/合规限制（不能随机拒绝给某些用户提供权益）；(2) 网络效应（社交产品的功能会影响未实验用户）；(3) 已上线的功能无法回退；(4) 样本量不足或时间紧迫。掌握观察数据因果推断是高级 PM 的必备能力。

**Q2：PSM 和 DID 各有什么优缺点？什么时候组合使用？**

PSM 的优势是直观、易实施，缺点是只能控制可观测变量。DID 的优势是可以控制不可观测但"不随时间变化"的偏差。组合使用时，PSM 先创造可比样本，DID 再消除时间固定偏差——这是观察研究中最稳健的方案之一，被称为"PSM-DID"。

**Q3：因果推断的结论如何向非技术的高管解释？**

用类比："就像比较吃药和没吃药的人——吃药的人可能本来就是更关注健康的人。我们做的是找到'健康意识相同的两组人'再比较。"然后用数字说话："表面差异 65%，消除偏差后是 18%——仍然有效，但效果是原来的三分之一。"

**Q4：如果混杂因素太多，PSM 匹配质量很差怎么办？**

匹配质量差的信号：匹配后协变量仍然不均衡（标准化差 > 10%）。解决方案：(1) 增加协变量；(2) 放宽匹配的容忍度（caliper），但接受更多偏差；(3) 改用加权方法（IPTW）替代匹配；(4) 如果实在无法控制，诚实报告局限性——"我们只能控制 X、Y、Z 因素，结果在 A-B 之间，不是精确值。"

**Q5：如何判断观察研究的结论是否"足够好"可以做决策？**

三个标准：(1) 多种方法的结果方向一致（都为正或都为负）；(2) 效果量的置信区间不包含 0；(3) 满足关键假设（平行趋势、匹配均衡等）。如果三个条件都满足，可以做出有条件的决策，并设置监控指标持续验证。

## 关键收获

1. **观察数据的因果推断是高级 PM 区别于初级 PM 的核心能力**——初级 PM 看"数字差异"，高级 PM 看"去偏后的真实效果"。
2. **没有银弹——每种方法都有假设和局限**，最可靠的做法是用多种方法交叉验证。
3. **诚实比精确更重要**——与其给出一个虚假的精确数字（65%），不如给出一个诚实的区间（15-25%）并说明假设。
4. **因果推断不是否定功能价值，而是校准预期**——把决策从"感觉很好"升级为"有条件的最优选择"。
