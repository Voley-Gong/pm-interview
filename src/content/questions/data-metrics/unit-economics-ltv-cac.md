## 面试场景

> 面试官：「你负责一款SaaS产品的增长，目前月新增客户1000人，获客成本逐月上升。投资人间你：按当前增长速度，公司的单元经济学是否健康？你会如何分析？」

这道题考察的是候选人对**单元经济学（Unit Economics）**的理解——即从单个用户/客户的视角，衡量收入与成本的关系。这是判断一个产品增长是否可持续的核心能力。

---

## 核心概念解析

### 什么是单元经济学？

单元经济学是指以**单个用户（或单个订单、单个客户）**为基本单位，分析其带来的收入与消耗的成本之间的关系。它回答一个根本问题：

**「每获取一个用户，我们最终赚的比花的多吗？」**

两个核心指标：

| 指标 | 全称 | 含义 |
|------|------|------|
| **CAC** | Customer Acquisition Cost | 获客成本：获取一个新用户/客户的平均成本 |
| **LTV** | Lifetime Value | 用户终身价值：一个用户在整个生命周期内贡献的净收入 |

### LTV 的计算

```
LTV = ARPU（每用户平均收入） × 用户生命周期 × 毛利率

其中：
- 用户生命周期 = 1 / 月流失率（Churn Rate）
- ARPU = 月均每用户收入
- 毛利率 = （收入 - 直接成本）/ 收入
```

**示例：**
- 月订阅费 ¥99，毛利率 80%，月流失率 5%
- 用户生命周期 = 1 / 0.05 = 20 个月
- LTV = 99 × 20 × 0.8 = ¥1,584

### CAC 的计算

```
CAC = 总获客投入（营销+销售） / 新增用户数

包含：广告费、内容营销、销售团队薪酬分摊、工具成本等
```

**示例：**
- 月营销+销售总投入 ¥500,000，新增 1000 用户
- CAC = 500,000 / 1000 = ¥500

## 核心框架：LTV/CAC 比值与 CAC Payback

### 1. LTV/CAC 比值（健康度标尺）

| 比值 | 状态 | 含义 |
|------|------|------|
| < 1 | ❌ 危险 | 每赚1元花超过1元，增长越快亏损越大 |
| 1 ~ 3 | ⚠️ 亚健康 | 能赚但利润薄，抗风险能力弱 |
| **≥ 3** | ✅ 健康 | 行业黄金标准，增长可持续 |
| > 5 | ⚠️ 投入不足 | 可能在过度保守地获客，错失增长机会 |

> **面试金句：**「LTV/CAC ≥ 3 是健康线，但 > 5 可能意味着获客投入不足，在竞争市场中被对手抢走份额。」

### 2. CAC Payback Period（回收周期）

```
CAC Payback = CAC / （ARPU × 毛利率）

含义：需要多少个月才能从单个用户身上收回获客成本
```

**健康标准：**
- SaaS 行业：≤ 12 个月为健康
- 电商：≤ 3 个月（因为复购周期短）
- B2B 企业级：≤ 18 个月可接受

### 3. 三指标联用判断框架

```
判断矩阵：
┌─────────────────┬──────────────┬───────────────┐
│                 │ Payback 短   │ Payback 长    │
├─────────────────┼──────────────┼───────────────┤
│ LTV/CAC ≥ 3     │ 🟢 快速增长  │ 🟡 增长但需注资│
│ LTV/CAC < 3     │ 🟠 模待验证  │ 🔴 危险模式   │
└─────────────────┴──────────────┴───────────────┘
```

## 实战案例

### 案例：在线教育平台的单元经济学诊断

**背景：** 某K12在线教育平台，月活50万，付费用户8万。

**数据：**
- 月均营销投入：¥2,000,000
- 月新增付费用户：2,000
- 月均客单价：¥200
- 月毛利率：60%
- 月流失率：15%

**计算：**
```
CAC = 2,000,000 / 2,000 = ¥1,000
ARPU = ¥200
生命周期 = 1 / 0.15 ≈ 6.67 个月
LTV = 200 × 6.67 × 0.6 ≈ ¥800

LTV/CAC = 800 / 1000 = 0.8 ❌
CAC Payback = 1000 / (200 × 0.6) ≈ 8.3 个月
```

**诊断结论：** LTV/CAC < 1，每获取一个用户亏损 ¥200。增长越快，亏损越大。

**优化方向：**
1. **降低 CAC**：优化投放渠道，提升自然流量占比（SEO、口碑）
2. **提升 LTV**：
   - 降低月流失率（15% → 8%），生命周期从 6.67 → 12.5 个月
   - 增加增值服务提升 ARPU
   - LTV 重算：200 × 12.5 × 0.6 = ¥1,500 → LTV/CAC = 1.5（改善但仍需努力）
3. **组合策略**：CAC 降到 ¥500 + LTV 提到 ¥1,500 → 比值 3.0 ✅

## 面试常见追问

### 追问1：「如果LTV/CAC目前只有1.5，但公司还在快速增长，你怎么看？」

**回答框架：**
- 首先区分**阶段**：早期产品可能处于「先规模化、后优化经济模型」策略（如 Uber、滴滴早期）
- 看**趋势**：LTV/CAC 是在改善还是恶化？如果逐月改善，说明飞轮效应开始显现
- 看**融资**：是否有足够资金支撑到转正？计算 runway
- 看**固定成本 vs 边际成本**：如果成本结构中固定成本占比高，规模效应会显著改善 LTV/CAC

### 追问2：「免费用户怎么算进 CAC？」

- CAC 计算时应包含**所有通过获客投入带来的用户**（包括免费用户）
- LTV 计算时只算**付费用户**
- 如果有免费→付费转化漏斗，应该用「有效付费 CAC」：

```
有效付费CAC = 总获客投入 / 新增付费用户数
```

### 追问3：「电商和SaaS的单元经济学有什么区别？」

| 维度 | SaaS | 电商 |
|------|------|------|
| LTV 计算周期 | 长（数月到数年） | 短（复购周期决定） |
| CAC 回收 | 慢（数月） | 快（首单即可） |
| 关键变量 | 流失率、续费率 | 复购率、客单价 |
| 危险信号 | CAC Payback > 18月 | 首单亏损且复购率低 |

## 标准回答框架

### STEP 1：定义指标，明确公式
> 「首先明确LTV和CAC的计算方式，确认数据口径一致。」

### STEP 2：计算核心比值
> 「LTV/CAC 比值看健康度，CAC Payback 看现金流压力。」

### STEP 3：行业对标
> 「将结果与行业基准对比，判断处于什么水平。」

### STEP 4：诊断根因
> 「LTV/CAC 不健康时，拆解到 ARPU、流失率、获客效率三个维度找根因。」

### STEP 5：给出优化方案
> 「从提升LTV（降流失、提客单价）和降低CAC（优化渠道、提升自然流量）两个方向给出具体措施。」

### STEP 6：风险提示
> 「注意过度优化LTV/CAC可能导致增长停滞，需要在健康度和增长速度间取平衡。」

## 关键收获

1. **LTV/CAC 是增长的「地基指标」**：不健康的单元经济学，增长越快死得越快
2. **三指标联用**：LTV/CAC + CAC Payback + 增长率，构成完整的商业健康度画像
3. **阶段不同，容忍度不同**：早期可以容忍不健康，但必须有明确的改善路径和时间表
4. **优化是系统性工程**：不是单纯砍预算或涨价，而是从获客效率、留存、变现三个维度协同优化
5. **面试中展示商业直觉**：不仅要算对数字，更要能从数字中读出业务故事和行动方向
