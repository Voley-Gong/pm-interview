## 面试场景

> 面试官："你负责一款在线教育App，最近发现主要竞品（一家头部玩家）因为改版引发了大量用户不满。老板让你去研究这些流失用户，看看能不能把他们拉过来。你会怎么做这个研究？"

这道题考察的不是"怎么抢竞品用户"的战术执行，而是你是否具备**系统化的竞品用户研究方法论**——从研究设计到洞察提取再到策略落地的全链路能力。

---

## 核心概念解析

### 为什么要研究竞品用户流失

传统竞品分析关注"对手有什么功能"，而**竞品用户流失研究**关注"对手的用户为什么失望"——后者往往揭示更真实的用户需求。

| 研究类型 | 关注点 | 典型方法 | 产出 |
|---------|--------|---------|------|
| 功能对标 | 对手做了什么 | 竞品拆解、功能矩阵 | 功能差距清单 |
| 用户感知 | 用户觉得怎么样 | NPS对比、评论分析 | 满意度差异 |
| 流失研究 | 用户为什么离开 | 深度访谈、社交聆听 | 痛点排序、机会地图 |

### 流失用户的三种类型

**1. 主动流失（Voluntary Churn）**
- 用户明确不满意，主动卸载/退订
- 价值最高：能清晰表达不满原因
- 研究难度：低（愿意吐槽）

**2. 被动流失（Involuntary Churn）**
- 因为竞品改版导致功能不可用/体验恶化而离开
- 价值高：能揭示"不可接受的体验底线"
- 研究难度：中（需要找到他们）

**3. 沉默流失（Silent Churn）**
- 逐渐减少使用，自然流失
- 价值中：原因往往模糊
- 研究难度：高（自己可能都说不清）

## 框架：竞品用户流失研究五步法

### Step 1 — 研究设计（Research Design）

**明确研究问题**：
- 主问题：竞品流失用户的核心痛点是什么？哪些是我们可以解决的？
- 子问题1：流失用户去了哪里？（竞品、替代品、还是干脆不用了）
- 子问题2：他们的迁移成本感知如何？
- 子问题3：他们对原产品最怀念的是什么？（这揭示了产品的"不可替代价值"）

**确定研究对象**：
- 近3个月内离开竞品的用户（记忆新鲜）
- 使用竞品超过3个月的用户（有足够体验深度）
- 涵盖不同付费层级（免费用户和付费用户的流失原因可能完全不同）

### Step 2 — 多渠道招募与数据采集

| 渠道 | 方法 | 优势 | 局限 |
|------|------|------|------|
| 社交媒体 | 监听竞品相关话题、超话、评论区 | 真实、自发、样本量大 | 表层信息，缺深度 |
| 应用商店评论 | 竞品App Store/应用商店差评追踪 | 结构化（有评分+文字） | 幸存者偏差（极端用户居多） |
| 社区/论坛 | 竞品用户社区、垂直社区帖子 | 深度讨论、对比观点 | 小众群体代表性有限 |
| 定性访谈 | 通过问卷筛选+1对1深度访谈 | 最深度的洞察 | 成本高、样本小 |
| 定量问卷 | 面向竞品用户的大规模问卷 | 可量化、可统计 | 回收率低、真实性存疑 |

**实战技巧——如何找到竞品流失用户**：
- 在相关社区发帖："最近从XX转到其他产品的同学，聊聊体验？"
- 利用用户画像定向投放研究邀请（如：教育行业群组中筛选）
- 对自己现有用户中"从竞品转来"的做回溯访谈（这是最容易忽略的宝藏）

### Step 3 — 数据分析与洞察提取

**定性数据分析——流失原因编码框架**：

```
L1 类别（大类）
├── 产品体验问题
│   ├── L2 子类：功能缺失 / 性能问题 / UI变化 / 学习成本
│   └── L3 具体描述：如"新版去掉了我最常用的离线下载"
├── 内容/服务质量问题
│   ├── L2 子类：内容质量下降 / 师资退步 / 价格上涨
├── 情感/信任问题
│   ├── L2 子类：隐私担忧 / 品牌信任崩塌 / 社区氛围恶化
└── 外部因素
    └── L2 子类：需求消失 / 经济原因 / 竞品更优
```

**定量数据——流失漏斗分析**：

| 触发事件 | 用户占比 | → 迁移去向 |
|---------|---------|-----------|
| 竞品改版公告 | 100% | — |
| 改版后首次体验 | 85% | 15%立即流失 |
| 尝试使用新版本 | 60% | 25%在1周内流失 |
| 持续使用2周后 | 35% | 25%在1月内流失 |
| 最终留存 | 20% | 80%流失 |

*（以上为示例数据结构，实际需通过研究填充）*

### Step 4 — 构建机会地图（Opportunity Map）

将流失原因按"痛感强度"和"我们解决能力"两个维度排序：

```
         高痛感
            │
    ┌───────┼───────┐
    │  II   │   I   │
    │ 快赢区│ 核心战│
    │       │       │
 ───┼───────┼───────┼─── 高解决能力
    │  III  │  IV   │
    │ 观望区│ 不碰  │
    │       │       │
    └───────┼───────┘
            │
         低痛感
```

**象限I（核心战场）**：痛感高 + 我们能解决 → 优先投入
**象限II（快赢机会）**：痛感高 + 我们已具备能力 → 立即行动，做精准获客
**象限III（观望储备）**：痛感低 + 能解决 → 中期规划
**象限IV（不碰）**：痛感低 + 不能解决 → 放弃

### Step 5 — 策略落地

**短期（1-2周）**：
- 针对核心战场痛点，设计"迁移引导方案"
- 制作对比内容（如"从XX到YY，你的学习记录可以这样迁移"）
- 在相关社区进行精准触达

**中期（1-2月）**：
- 产品层面补齐竞品流失用户最渴望的功能
- 建立竞品流失用户专属onboarding路径（降低迁移摩擦）

**长期（季度）**：
- 持续监听竞品用户满意度，建立预警机制
- 将竞品流失研究制度化——每季度做一轮

## 面试常见追问

### 追问1："你怎么确保找到的竞品流失用户是真实的，而不是为了奖励随便填的？"

**质量控制手段**：
- **行为验证问题**：要求描述具体的产品使用细节（如"新版课程列表页面你第一眼看到了什么？"），真实用户能回答，编造者会含糊
- **交叉验证**：对比应用商店评论与访谈内容的一致性
- **渐进式深入**：先问开放问题，逐步深入到操作细节层面
- **适当激励但不过度**：激励过高会吸引"职业问卷填写者"

### 追问2："如果研究发现竞品流失用户的主要原因是'价格太高'，我们该不该打价格战？"

**分析框架**：
1. 区分"价格敏感型流失"和"价值感知不足型流失"——用户说"太贵了"可能意味着"不值这个价"
2. 如果确实是价格问题：考虑**分层定价**而非简单降价（如推出免费基础版+付费高级版）
3. 关注LTV而非单次转化：低价获客但留存差，不如精准获取高价值用户
4. 价格战是最后手段——一旦开始，很难回头

### 追问3："这个研究结论怎么转化为具体的产品决策？给我一个例子。"

**示例转化链路**：
```
研究发现：竞品用户因"新版去掉了学习数据导出功能"而流失
→ 洞察：用户对"数据所有权"有强烈需求
→ 产品决策：在onboarding中加入"一键导入你的学习记录"功能
→ 获客策略：在竞品社区传播"你的学习数据你做主"的信息
→ 指标：迁移功能使用率、竞品来源用户7日留存
```

## 标准回答框架

```
1. 重构问题（20秒）
   "不只是'怎么抢用户'，而是'理解他们为什么不满意，以及我们能提供什么不同的价值'"

2. 研究设计（40秒）
   研究问题 → 对象定义 → 多渠道采集 → 定性+定量结合

3. 分析框架（30秒）
   流失原因编码 → 痛感×解决能力机会地图

4. 策略落地（30秒）
   短期精准触达 + 中期产品补齐 + 长期监听机制

5. 补充思考（20秒）
   研究伦理：不欺骗、不操纵，靠产品力赢得用户
```

## 关键收获

1. **竞品用户流失研究的核心价值**：从对手的错误中学习，比从自己的成功中学习更高效
2. **多渠道交叉验证**是研究质量的关键——任何单一渠道都有偏差
3. **机会地图（痛感×解决能力）**是将研究洞察转化为产品决策的核心工具
4. **迁移成本感知**是影响竞品用户切换的关键心理因素——降低迁移摩擦比打价格战更有效
5. **研究制度化**比单次研究更重要——竞品用户研究应该是持续的市场雷达，而非一次性行动
6. **技术转PM的优势**：能更深入理解竞品改版背后的技术限制，判断"哪些痛点是竞品改不回来的"（如架构限制导致的功能取舍）
