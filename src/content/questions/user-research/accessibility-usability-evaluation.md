## 面试场景

> 面试官：「我们的一款SaaS产品的活跃用户增长遇到了瓶颈。数据显示，注册到激活的漏斗中，35岁以上用户的流失率是年轻用户的2.3倍。你会如何研究这个问题？在你的研究方案中，你会考虑哪些常被忽略的用户群体？」

这道题的深层考点不是"你会不会做用户研究"，而是**你的研究视野是否足够包容**——你能否超越"标准用户画像"，看到那些被默认排除在外的群体。无障碍设计（Accessibility）和包容性研究（Inclusive Research）正是区分初级PM和资深PM的分水岭。

---

## 一、核心概念解析：无障碍设计不只是"给残障人士做适配"

### 1.1 无障碍设计的真实范围

很多人把Accessibility等同于"给盲人做读屏"或"给轮椅用户做坡道"，这是严重的认知窄化。实际上，无障碍覆盖的人群远比想象中广：

| 障碍类型 | 永久性 | 临时性 | 情境性 |
|---------|--------|--------|--------|
| 视觉 | 盲/低视力 | 眼部术后 | 强光下看屏幕 |
| 听觉 | 聋/重听 | 中耳炎 | 嘈杂环境 |
| 运动能力 | 截肢/运动障碍 | 手臂骨折 | 抱着婴儿操作 |
| 认知 | 阅读障碍/ADHD | 严重疲劳 | 压力下操作 |
| 语言 | — | — | 非母语用户 |

**关键洞察：世界上约有15%的人有某种形式的残障，但几乎100%的人在某些情境下都会遇到"类残障"的交互困难。**

### 1.2 包容性用户研究 vs 传统用户研究

| 维度 | 传统研究 | 包容性研究 |
|------|---------|-----------|
| 招募标准 | "18-35岁、智能手机用户" | 刻意招募多样化参与者 |
| 研究假设 | "用户都能看到这个按钮" | "如果用户看不到怎么办？" |
| 边缘案例 | 视为特殊情况 | 视为设计改进的机会 |
| 研究环境 | 实验室标准环境 | 用户的真实使用环境 |
| 数据解读 | 看平均值和主流趋势 | 关注分布尾部和异常值 |

### 1.3 适老化是无障碍的重要子集

中国的老龄化趋势使得适老化已经从"道德选择"变为"法律要求"（工信部适老化改造指引）。但好的适老化设计不是简单"字放大"，而是需要系统性的研究：

- 认知负荷管理（信息密度降低、操作步骤简化）
- 运动能力退化（更大的点击区域、更宽容的误触处理）
- 视觉变化（不只是字体大小，还有对比度、色彩辨识）
- 数字素养差距（对"常见交互模式"的熟悉程度不同）

## 二、评估框架：无障碍成熟度模型（A11y Maturity Model）

### Level 0：无意识
- 没有人想过无障碍这件事
- 设计稿只考虑了"标准"用户
- 出了投诉才临时修补

### Level 1：合规驱动
- 知道WCAG的存在，但只是为了应付检查
- 无障碍是"额外工作"，不是设计的一部分
- 测试阶段才考虑，通常只是走查清单

### Level 2：融入流程
- 设计阶段就有无障碍检查点
- 用研招募包含多样化用户
- 开发有自动化无障碍测试（如axe、Lighthouse）
- PM在需求评审时会问"这个功能对XX用户的影响是什么？"

### Level 3：战略级优势
- 无障碍是产品差异化优势
- 主动超越WCAG标准，追求"极致可用"
- 有专门的无障碍设计系统组件
- 定期发布无障碍报告，接受外部监督

### Level 4：包容性原生
- 无障碍不是独立议题，而是"好设计"的同义词
- 所有用户研究默认包含多样化参与者
- 产品体验对所有用户群体都有可度量的高质量表现

## 三、实战方法论：包容性用户研究四步法

### Step 1：扩展招募——谁在你的研究盲区里？

**标准招募清单的反思：**
```
常规招募：18-40岁，智能手机用户，每周使用互联网5小时以上

包容性补充：
□ 50岁以上用户（适老化需求）
□ 非母语用户（语言障碍）
□ 低收入用户（设备性能差、网络不稳定）
□ 有残障的用户（视障、听障、运动障碍、认知障碍）
□ 数字新手（最近1-2年才开始使用智能设备）
□ 非主流设备用户（低端安卓机、旧版浏览器）
```

**实操建议：** 不需要每次研究都覆盖所有群体，但每个季度至少有一次专门针对"非主流用户"的研究。

### Step 2：研究方法适配——让方法服从于人，而非人迁就方法

| 研究方法 | 传统做法 | 包容性改造 |
|---------|---------|-----------|
| 可用性测试 | 到实验室来 | 去用户的环境中 |
| 问卷调查 | 在线表单 | 提供电话/语音选项 |
| 用户访谈 | 30分钟视频通话 | 灵活时长、异步文字选项 |
| 眼动追踪 | 固定设备校准 | 结合回顾性口头报告 |

**关键原则：** 如果某种研究方法本身就把某类用户排除在外，那研究结论的代表性就需要质疑。

### Step 3：任务设计——从"能不能用"到"好不好用"

不要只测"用户能否完成任务"，要测：

- **效率差距**：不同用户群体完成同一任务的时间差
- **错误率差异**：哪些群体更容易出错？什么类型的错误？
- **求助频率**：用户在哪些步骤需要外部帮助？
- **情绪曲线**：过程中何时出现挫败、焦虑、困惑？
- **放弃意愿**：在什么节点用户想要放弃？

### Step 4：洞察合成的"多样性透镜"

在分析用户研究数据时，用三个透镜叠加审视：

```
透镜1：主流体验 — 大多数用户的核心路径是否流畅？
透镜2：边缘体验 — 弱势群体遇到的障碍是什么？是否可修复？
透镜3：通用收益 — 解决边缘问题的改进，对主流体验是否有正向溢出？
```

**经典案例：** 最初为视障用户设计的"语音读屏"功能，现在成为所有用户在驾驶场景下的刚需。为认知障碍用户简化的操作流程，提升了所有用户的效率。这就是**"为边缘设计，惠及所有人"（Curb Cut Effect，路沿斜坡效应）**。

## 四、实战案例：SaaS产品高龄用户流失分析

回到开头的面试题，一个结构化的回答：

### 4.1 研究设计

**阶段一：定量诊断（1周）**
- 按年龄分层重构漏斗，定位35+用户具体在哪一步流失
- 与行业基准对比，判断是否是普遍现象
- 做同期群分析，看不同获客渠道的35+用户流失率差异

**阶段二：定性深挖（2周）**
- 招募8-10位35-55岁的流失用户做深度访谈
- 招募5位55岁以上的潜在用户做可用性测试
- 对照组：招募5位25-30岁的流失用户，排除年龄因素

**阶段三：方案验证（1周）**
- 基于定性洞察设计3个改进方案
- 用原型做A/B测试，看是否缩小年龄分层间的流失率差距

### 4.2 可能的研究发现（假设性）

| 发现 | 根因 | 设计改进 |
|------|------|---------|
| 注册步骤中"邮箱验证"流失率高 | 高龄用户不习惯切换App查邮箱 | 支持手机号注册或一键登录 |
| 激活引导的视频教程完播率低 | 语速快、字幕小、无暂停提示 | 可控速、大字幕、图文版替代 |
| 核心功能的入口图标认知困难 | 使用了年轻群体熟悉的icon（如👍），高龄用户不理解 | 增加文字标签 |
| 设置页面的信息密度过高 | 视觉搜索效率下降 | 分组折叠、渐进式展开 |

### 4.3 长期建议

- 建立用户画像的"年龄多样性"标准
- 设计系统中加入适老化组件规范
- 季度可用性测试中纳入至少20%的35+用户
- 跟踪一个新指标：**「年龄分层的任务完成率差异」**，作为包容性设计的产品健康度指标

## 五、面试常见追问

### 追问1：「WCAG 2.2的核心原则是什么？你如何在产品中落地？」

参考回答：WCAG的四大原则是**POUR**——Perceivable（可感知）、Operable（可操作）、Understandable（可理解）、Robust（健壮性）。落地方式：在设计系统中内置WCAG合规检查、在Code Review中加入无障碍审查、使用自动化工具（axe、Pa11y）做持续检测。但工具检查只能覆盖约30%的标准，剩余70%需要人工评估。

### 追问2：「无障碍改造和业务目标冲突时怎么取舍？」

参考回答方向：
- 用数据说话——15%的市场不可忽视，适老化是政策趋势
- 路沿斜坡效应——无障碍改进的收益溢出到主流用户
- 合规风险——不做可能面临法律风险（ADA诉讼在国内也逐渐增多）
- 分阶段策略——优先修复影响面最大的障碍，纳入技术债管理

### 追问3：「你如何说服团队投入资源做无障碍，而不是做新功能？」

参考方向：
- 把无障碍重新定义为"扩大可达市场"，而非"额外成本"
- 用竞品对比——如果竞品做得更好，我们就在流失用户
- 与品牌价值挂钩——包容性是品牌成熟度的体现
- 巧妙的MVP策略——先做ROI最高的改进（如对比度、键盘导航），逐步深入

## 六、标准回答框架

```
无障碍评估回答 =
    1. 问题重构：不只是"高龄用户流失"，而是"包容性设计缺口"
  + 2. 研究设计：定量诊断 → 定性深挖 → 方案验证
  + 3. 框架引用：A11y成熟度模型 / WCAG POUR原则 / 路沿斜坡效应
  + 4. 数据支撑：15%残障人口 + 35+适老化趋势
  + 5. 落地策略：从研究洞察 → 设计系统 → 产品指标的全链路
```

## 七、关键收获

1. **无障碍设计不是"特殊需求"，而是"好设计"的标准之一** —— 一个对视障用户友好的界面，对强光下的所有用户都友好。

2. **包容性研究的ROI不是零和博弈** —— 路沿斜坡效应证明：为边缘群体做的改进，几乎总是会带来主流体验的提升。

3. **用户研究的盲区往往不是"方法不对"，而是"招募不对"** —— 如果你只研究"标准用户"，你永远只能发现标准问题。

4. **无障碍成熟度是组织能力的体现** —— Level 0到Level 4的差距，不是技术差距，而是认知差距和流程差距。PM是推动这个升级的关键角色。

5. **在面试中展示包容性视野，是区分"执行型PM"和"战略型PM"的有效信号** —— 它展示了你的同理心、系统性思维和商业敏感度的交叉点。
