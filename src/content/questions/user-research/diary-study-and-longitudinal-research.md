## 面试场景

> 面试官：你们做了一个健身 App，用户注册时的热情很高，但 30 天后的留存率只有 12%。你做了几次用户访谈，用户都说"我确实想坚持锻炼，但就是没时间"。你知道这句话不够——你需要理解用户从"充满热情"到"放弃使用"这段时间里到底发生了什么。你会怎么设计研究来回答这个问题？

这道题考察的是你对**时间维度上的用户研究**的理解。传统的用户访谈和可用性测试是"快照式"的——捕捉某一时刻的状态。但很多产品问题（留存、习惯养成、使用频率下降）本质上是**时间问题**，需要**纵向研究方法**才能回答。

## 核心概念：为什么需要纵向研究

### 横截面研究 vs 纵向研究

| 维度 | 横截面研究 | 纵向研究 |
|------|------------|----------|
| 时间 | 某一时刻 | 一段时间（天/周/月） |
| 回答的问题 | 用户"现在"怎样 | 用户"如何变化" |
| 典型方法 | 问卷、访谈、可用性测试 | 日记研究、体验采样、面板分析 |
| 数据类型 | 状态 | 过程、轨迹 |
| 适合场景 | 比较不同群体 | 理解变化、习惯、遗忘 |

### 用户行为的时间陷阱

1. **回忆偏差（Recall Bias）**：用户事后回忆的行为往往不准确。"你上周为什么没用？"→ 用户会编造一个合理的解释，但不是真实原因。
2. **意图-行为鸿沟（Intention-Behavior Gap）**：用户说"我打算每天用"时是真心的，但意图和行为之间存在巨大的执行鸿沟。
3. **新鲜效应衰减（Novelty Decay）**：新产品的兴奋感会随时间衰减，只有纵向研究才能区分"真价值"和"新鲜感驱动"的使用。

## 方法论：四种纵向研究方法

### 方法一：日记研究（Diary Study）

用户在一段时间内定期记录自己的行为、感受和场景。

**操作流程：**
1. **招募**：15-30 名参与者（定性日记研究不需要大样本）
2. **工具选择**：纸笔日记、App 提醒、微信群报备、在线表单
3. **频率设计**：
   - 事件触发式：用户做某事时记录（如"每次打开健身 App 时拍照"）
   - 时间触发式：每天固定时间记录（如每晚 8 点推送提醒）
   - 混合式：关键事件记录 + 每日总结
4. **持续时间**：通常 1-4 周，根据产品类型调整
5. **激励机制**：每日打卡奖励 + 完成全周期额外奖励

**日记内容设计模板：**
```
📅 日期：____
⏰ 现在的时间：____
📍 我在哪里：____
😊 我现在的心情：😀 😐 😞（选一个）
💪 我今天锻炼了吗？ □ 是 □ 否
→ 如果是：用了 App 吗？ □ 是 □ 否
→ 如果没用 App：为什么？
   □ 忘了打开
   □ 觉得太麻烦
   □ 锻炼方式不需要 App
   □ App 里没有我今天想做的运动
   □ 其他：____
→ 如果用了 App：体验如何？（1-5 分）____
一句话描述今天的感受：____________
```

### 方法二：体验采样方法（ESM, Experience Sampling Method）

在用户使用过程中随机"弹出"简短问卷。

**特点：**
- 比日记研究侵入性更低（每次只需 10-30 秒）
- 时机更精确（捕获当下感受）
- 需要在产品中嵌入或使用第三方工具
- 适合已有一定规模的产品

**示例：**
```
[推送通知/弹窗] "你现在感受如何？"
😊 棒极了 → 追踪什么让你满意
😐 还行   → 不做深入追问
😞 不好   → 追踪痛点所在
```

### 方法三：被动行为追踪（Passive Behavioral Tracking）

通过数据分析追踪用户行为的时间序列，不做主动干预。

**适用指标：**
- 登录频率的变化曲线
- 功能使用从探索期到稳定期的迁移
- 关键事件触发的时间间隔

### 方法四：回溯性面板研究（Retrospective Panel）

招募不同使用阶段的用户，通过深度访谈重构其完整使用历程。

**特点：**
- 不需要实时追踪（成本较低）
- 受回忆偏差影响（需要通过行为数据交叉验证）
- 适合探索性研究阶段

## 实战框架：健身 App 日记研究设计方案

### 研究问题
用户从注册到放弃的 30 天内，行为和心理经历了哪些变化阶段？

### 研究设计

**参与者招募：**
- 20 名刚注册的新用户（注册 3 天内）
- 覆盖三类：高频运动者（每周 3 次以上）、低频运动者（每周 1-2 次）、几乎不运动者
- 年龄/性别/职业尽量分散

**研究周期：** 30 天

**数据收集方式：**

| 时间点 | 方法 | 目的 |
|--------|------|------|
| 第 0 天 | 入组访谈（30分钟） | 了解初始动机、预期、过往经历 |
| 第 1-7 天 | 每日日记（晚上推送） | 捕获"蜜月期"的行为和心理 |
| 第 7 天 | 微访谈（15分钟） | 第一周回顾：匹配预期吗？ |
| 第 8-21 天 | 隔日日记 | 捕获"新鲜感消退"后的变化 |
| 第 14 天 | 行为数据提取 | 对比日记与实际使用数据 |
| 第 22-30 天 | 每周一次日记 | 捕获"放弃或坚持"的关键转折 |
| 第 30 天 | 出组访谈（45分钟） | 完整回顾：什么改变了？为什么？ |

### 预期发现（假设性）

通过日记研究，你可能发现以下模式：

**模式1：动机转型失败**
- 第 1-5 天：靠"新鲜感"驱动使用
- 第 6-12 天：新鲜感消退，开始靠"自律"维持
- 第 13-20 天：自律耗竭，使用频率下降
- 第 21-30 天：彻底放弃或沦为偶尔打开
→ **核心洞察**：App 没有帮用户完成从"外部动机（新鲜感）"到"内部动机（成就感）"的转型

**模式2：不可见进度**
- 用户期望 2 周内看到身体变化
- 实际身体变化需要 4-8 周
- 第 2 周用户感觉"练了也没变化"→ 动力骤降
→ **核心洞察**：App 需要在早期提供"微成就感"（如运动时长累计、卡路里目标完成）来弥补身体变化滞后

**模式3：社交断裂**
- 第 1 周积极分享运动成果到朋友圈
- 无人互动后逐渐停止分享
- 停止分享与停止使用高度相关
→ **核心洞察**：社交反馈是关键留存驱动，但当前分享机制未能形成正循环

## 面试常见追问

### 追问1："日记研究的数据怎么分析？每天那么多条记录。"

分析框架：
1. **时间轴编码**：将每个用户的行为按时间排列，找到"转折点"
2. **模式聚类**：将相似的行为轨迹分组（如"稳定型""衰减型""波动型"）
3. **触发分析**：在转折点前后寻找触发事件（如"第一次没达到目标后的情绪变化"）
4. **交叉验证**：将日记数据与 App 行为数据对比，找出"说的"和"做的"之间的差异
5. **旅程图重构**：基于数据为每种典型模式绘制用户旅程图

### 追问2："用户不愿意每天写日记怎么办？"

降低参与门槛的策略：
- **极简化**：每天只需选 3 个选项 + 1 句话（总耗时 < 1 分钟）
- **多样化输入**：支持语音、拍照、emoji（降低书写负担）
- **智能提醒**：在用户最可能回应的时间推送（基于前期行为数据）
- **激励设计**：阶梯式奖励（7天/14天/30天分别给予不同奖励）
- **反馈闭环**：给用户看到自己积累的数据可视化（让日记本身变得有价值）

### 追问3："日记研究的样本量怎么定？"

- 定性日记研究：15-30 人（目标是发现模式，不是统计显著性）
- 定量日记研究（结合数据分析）：50-200 人
- 关键不是样本量，而是**参与者多样性**（覆盖不同用户类型）和**完成率**（通常 60-70% 为合格）

### 追问4："你怎么区分'日记研究'和'问卷'的区别？"

| 维度 | 问卷 | 日记研究 |
|------|------|----------|
| 时间 | 一次性 | 多次持续 |
| 数据 | 态度/偏好 | 行为+情境+心理变化 |
| 偏差 | 回忆偏差大 | 实时记录偏差小 |
| 分析 | 群体比较 | 个体轨迹+群体模式 |

## 标准回答框架（面试时）

```
1. 明确研究问题（30秒）
   "我要回答的不是'用户为什么不留存'，而是'用户从兴奋到放弃的过程中经历了什么'"

2. 选择方法并论证（1分钟）
   "我会用日记研究，因为……"
   → 解释为什么横截面方法不够
   → 解释日记研究的优势

3. 研究设计（1分钟）
   "我的设计是……"
   → 招募、周期、数据收集方式
   → 展示结构化思维

4. 预期产出（30秒）
   "我预期发现……"
   → 展示你对用户行为的假设
   → 说明发现如何转化为产品行动

5. 局限性认知（30秒）
   "这个方法的局限是……"
   → 样本量、参与者偏差、霍桑效应
   → 如何通过其他方法补充
```

## 关键收获

1. **时间维度是用户研究的盲区**——大多数研究方法是"快照"，但很多产品问题是"电影"
2. **日记研究的核心价值是捕捉过程**——不是"用户做了什么"，而是"用户如何从做变为不做"
3. **回忆偏差是用户研究的头号敌人**——用户说的"原因"往往是事后建构的合理化叙述，不是真实的因果
4. **定性纵向研究 + 定量行为数据 = 最佳组合**——日记告诉你"为什么"，行为数据告诉你"实际上发生了什么"
5. **研究设计要尊重用户**——日记研究的侵入性较高，精心设计体验是对参与者的尊重，也直接影响数据质量
