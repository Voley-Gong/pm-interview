# 如何为新产品设计定价策略？

## 面试场景

> 面试官："你正在负责一款 B2B SaaS 项目管理工具，目标客户是 50-200 人的中小企业。市场上已有竞品免费版和低价方案。你会如何设计定价策略？需要考虑哪些因素？"

这类题目考察的是：**你能否系统性地思考"定价"这个产品决策——它不是运营的事，而是产品策略的核心组成部分**。

---

## 一、为什么定价是 PM 的核心能力？

### 定价的本质

很多技术转 PM 的候选人会忽略一个事实：**定价就是产品的一部分**。

- 定价决定了**谁会成为你的用户**（用户筛选）
- 定价决定了**产品能否持续存活**（商业模式闭环）
- 定价体现了**你对自身产品价值的信心**（价值定位）

> 一句话：定价是你把"产品价值"翻译成"市场语言"的过程。

### 技术背景 PM 的常见误区

| 误区 | 表现 | 正确认知 |
|------|------|----------|
| 越便宜越好 | 试图用低价打败所有人 | 低价不是策略，价值才是 |
| 定价 = 成本 + 利润 | 只算成本加价 | 成本定价忽略了用户支付意愿 |
| 免费是最好的获客 | 一上来就免费 | 免费用户不等于真实需求验证 |
| 定价一旦定就不改 | 害怕调价 | 定价需要随产品阶段迭代 |

## 二、四大定价方法论

### 1. 成本加成定价（Cost-Plus Pricing）

**公式：** 价格 = 单位成本 × (1 + 目标利润率)

**适用场景：** 制造业、传统零售、基础设施类产品

**优点：** 计算简单，利润有保障

**缺点：** 完全忽略用户支付意愿和竞争环境

> 在软件/互联网产品中几乎不推荐作为主要方法，但可以作为**价格下限参考**。

### 2. 竞争导向定价（Competitive Pricing）

**方法：** 参照竞品价格，定在相同/略高/略低位置

**适用场景：** 成熟市场、同质化较高的产品

**优点：** 快速定位，市场接受度高

**缺点：** 陷入价格战；无法体现差异化价值

**实操技巧：** 制作竞品价格矩阵

```
| 竞品      | 免费版      | 入门版    | 专业版     | 企业版      |
|-----------|-------------|-----------|------------|-------------|
| Jira      | 10用户免费  | $1.53/人  | $8.15/人   | 定制报价    |
| Asana     | 基础免费    | $10.99/人 | $24.99/人  | 定制报价    |
| Monday    | 基础免费    | $8/人     | $10/人     | $16/人      |
| 我们的产品 | ?           | ?         | ?          | ?           |
```

### 3. 价值定价（Value-Based Pricing）

**核心思想：** 价格 = 用户感知到的价值 × 支付意愿系数

**这是 PM 最应该掌握的定价方法。**

**操作步骤：**

1. **量化用户价值：** 你的产品帮用户省了多少钱/时间？赚了多少钱？
2. **确定价值分配比：** 通常产品获取所创造价值的 10%-30%
3. **设定价格锚点：** 用高端套餐锚定价值感知

**案例：**
- 你的项目管理工具帮一个 50 人团队每月节省 20 小时沟通成本
- 假设人力成本 100 元/小时 → 月节省 2000 元
- 取 20% 价值分配 → 定价 400 元/月（即 8 元/人/月）

### 4. 动态定价 / 分层定价（Dynamic / Tiered Pricing）

**形式：**
- **按量计费**（API 调用次数、存储空间）
- **按席位计费**（用户数 × 单价）
- **按功能分层**（Free / Pro / Enterprise）
- **按时段定价**（高峰溢价、促销折扣）

## 三、SaaS 定价的实战框架：五步定价法

### Step 1：明确商业目标

| 目标 | 定价倾向 | 典型阶段 |
|------|----------|----------|
| 快速获客 | 低价/免费 | 早期验证 PMF |
| 验证付费意愿 | 中高价 | PMF 后增长期 |
| 最大化营收 | 价值定价+分层 | 成熟期 |
| 抵御竞争 | 有策略的低价 | 防御期 |

### Step 2：理解用户支付意愿（WTP）

**如何测量 WTP：**
- **Van Westendorp 价格敏感度测试（PSM）**：问用户四个问题
  1. 什么价格让你觉得太贵，不会买？
  2. 什么价格让你觉得有点贵，但还会考虑？
  3. 什么价格让你觉得划算？
  4. 什么价格让你觉得太便宜，怀疑质量？
- **竞品替代法**：用户目前用什么替代方案？花多少钱？

### Step 3：设计价格结构

**经典 SaaS 三层定价：**

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   免费版     │    │   专业版     │    │   企业版     │
│  Free       │    │  Pro        │    │  Enterprise │
│             │    │             │    │             │
│ • 基础功能   │ →  │ • 全功能     │ →  │ • 定制化     │
│ • 限制用量   │    │ • 优先支持   │    │ • SLA保障    │
│ • 社区支持   │    │ • 团队协作   │    │ • 专属客户经理│
│             │    │             │    │             │
│ 获客引流     │    │ 主要收入来源  │    │ 高利润贡献   │
└─────────────┘    └─────────────┘    └─────────────┘
```

**设计要点：**
- 免费版的目的是**降低体验门槛**，不是送礼
- Pro 版和 Enterprise 版之间要有**明显的价值跳跃**
- 利用"诱饵效应"：三档中间的通常是主推

### Step 4：设定初始价格

**经验法则：**
- B2B SaaS：每个用户每月 10-50 美元为常见区间
- 价格用"奇数"（如 99、199）比"整数"转化更好（心理学定价）
- **年付折扣** 15%-20% 是行业标准

### Step 5：持续迭代

定价不是一锤子买卖。你需要定期（每 6-12 个月）评估：
- 各层级的转化率和升级率
- 用户的价格抱怨率
- 竞品价格变动
- 产品新增功能是否值得调价

## 四、面试常见追问

### Q1："如果用户觉得你的产品比竞品贵，怎么办？"

**回答框架：**
1. 先确认：是**价格问题**还是**价值传达问题**？（80% 的情况是后者）
2. 如果是价值传达：优化产品叙事、增加 ROI 计算器、做案例营销
3. 如果确实是价格问题：评估是否需要推出**入门版**降低门槛，而非整体降价

### Q2："免费 vs 付费，你怎么决定？"

**决策框架：**
- **用免费版**：如果你的产品有网络效应（用户越多越有价值）、获客成本高、竞品已免费
- **直接收费**：如果你的产品面向企业、有明确 ROI、需要服务支持
- **混合模式**：Freemium（免费+付费），但必须确保免费版不会"太好用"

### Q3："如果要做促销折扣，你会怎么做？"

**原则：**
- 促销是**战术**，不是战略
- 永远不要在**价值传达不充分**时做折扣（等于自杀定价）
- 折扣要有明确理由（限时、新用户、年度套餐），不能无理由打折
- 折扣结束后要观察**续费率**，判断用户是冲着价值来的还是冲着折扣来的

## 五、标准回答框架

遇到定价策略面试题，按以下结构组织回答：

```
1. 先问澄清问题 → 商业目标是什么？目标客户是谁？当前处于什么阶段？
2. 分析市场 → 竞品怎么定价？用户替代方案的成本是多少？
3. 选择定价方法 → 基于产品特点选 1-2 种方法（通常价值定价+分层）
4. 设计价格结构 → 几个层级？每个层级的边界是什么？
5. 给出初始价格 → 带具体数字，有理有据
6. 验证方法 → 如何测试这个定价是否合理？什么信号说明要调？
```

## 六、关键收获

1. **定价是产品决策，不是运营决策**——PM 必须深度参与
2. **价值定价 > 竞争定价 > 成本定价**——始终从用户感知价值出发
3. **三层定价是 SaaS 的经典范式**——免费引流、专业创收、企业溢价
4. **定价需要持续迭代**——不调价的产品要么被低估，要么被淘汰
5. **技术转 PM 的优势**：你能更好地理解产品的边际成本结构，避免"免费到底"的天真方案

> 最后记住：**如果你不知道自己的产品值多少钱，用户也不会知道。** 定价的第一步，是你自己对产品价值有清晰的信念。
