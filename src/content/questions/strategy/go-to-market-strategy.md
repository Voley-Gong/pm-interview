## 什么是 Go-To-Market 策略

Go-To-Market（GTM）策略是**将产品从"做出来了"推进到"用户在用、并且在付费"的完整执行计划**。它不是一份发布清单，而是回答四个战略问题：

1. **卖给谁**（目标市场与定位）
2. **怎么卖**（渠道与销售模式）
3. **卖什么**（定价与包装）
4. **怎么推**（市场推广与发布节奏）

面试官通过这道题考察：
- 你是否有**从产品到商业的全链路思维**（不只是做功能）
- 你能否**系统性规划产品发布**（而不是上线了就完事）
- 你是否理解**不同产品类型的 GTM 差异**（B2B vs B2C vs PLG）
- 你能否在**资源有限时做出取舍**（GTM 不是面面俱到，而是聚焦突破）

## GTM 策略六步框架

### 第一步：市场定义与目标用户

**核心问题：这个产品到底卖给谁？第一批用户是谁？**

#### 目标市场分层

```
TAM（Total Addressable Market）—— 总可服务市场
  ↓
SAM（Serviceable Addressable Market）—— 可服务市场
  ↓
SOM（Serviceable Obtainable Market）—— 可获取市场（3年内实际能拿到的份额）
  ↓
Beachhead Market —— 滩头堡市场（第一个要拿下的细分市场）
```

**Beachhead 策略**（来自《穿越冰川》Crossing the Chasm）：

不要一上来就打大市场。先选一个**足够小、足够痛、你能赢**的细分市场作为滩头堡，拿下后再向相邻市场扩张。

| 产品 | Beachhead 市场 | 后续扩张 |
|------|---------------|----------|
| Facebook | 哈佛大学生 | → 常春藤 → 全美大学 → 全球 |
| Slack | 科技创业团队 | → 中型科技公司 → 大企业 |
| Notion | 产品经理/设计师 | → 创业团队 → 企业 |
| Zoom | 中小企业远程会议 | → 大企业 → 个人用户 |

#### 用户画像精确定义

好的 GTM 不只说"25-35岁白领"，而是精确到：

```
目标用户：25-30岁一线城市产品经理/设计师
痛点：团队知识分散在多个工具中，找不到历史文档
使用场景：每周需要回顾项目文档、撰写周报
购买决策者：本人（个人工具）或团队Leader（团队采购）
支付意愿：愿意为效率工具月付 30-80 元
信息获取渠道：即刻、ProductHunt、同事推荐、B站
```

### 第二步：定位与价值主张

**核心问题：用户为什么要选你，而不是竞品或"不做"？**

#### 价值主张画布（Value Proposition Canvas）

```
【用户端】
- Jobs：用户要完成什么任务？
- Pains：在这个过程中什么让用户痛苦？
- Gains：什么会让用户感到满意？

【产品端】
- Products/Services：我们提供什么？
- Pain Relievers：我们如何缓解用户的痛苦？
- Gain Creators：我们如何创造用户的价值？

→ 匹配点就是核心价值主张
```

#### 定位声明模板

```
对于 [目标用户]，我们的产品是一个 [品类]，
它能帮助用户 [核心价值]，
不同于 [主要竞品]，我们的独特优势是 [差异化]。
```

**案例：**
> 对于 5-50 人的创业团队，Notion 是一个一体化工作空间，
> 能帮助团队统一管理文档、项目、知识库，
> 不同于 Confluence（重、贵、独立部署），Notion 的优势是极简灵活、无需部署、免费起步。

### 第三步：定价与商业模式

**定价不是"随便定个价"，而是产品定位的核心组成部分。**

#### 常见定价模式对比

| 定价模式 | 适用产品 | 优势 | 风险 |
|----------|----------|------|------|
| **免费增值（Freemium）** | 工具、社交、内容 | 低门槛获客、自传播 | 免费用户成本高 |
| **免费试用（Free Trial）** | SaaS、B2B | 高转化率 | 门槛较高 |
| **订阅制** | 内容、SaaS、工具 | 可预期收入 | 流失风险 |
| **按量付费** | API、云服务、基础设施 | 公平透明 | 收入不可预期 |
| **一次性买断** | 工具类 App | 简单 | 无复购 |
| **交易佣金** | 平台、电商 | 与用户利益一致 | 需要规模效应 |

#### 定价心理学技巧

| 技巧 | 说明 | 案例 |
|------|------|------|
| **锚定效应** | 先展示高价选项让其他选项显得划算 | 三列定价：$99 / $49 / $19（中间为主推） |
| **诱饵选项** | 加入一个不卖的高价选项引导选择 | 经济舱 $200 / 商务舱 $800 / 头等舱 $1500（商务舱看起来合理了） |
| **整数定价 vs 尾数定价** | 高端用整数，大众用尾数 | $100 vs $99（前者显高端，后者显便宜） |
| **年付折扣** | 年付比月付便宜 15-20% | 提升留存 + 提前回款 |

### 第四步：渠道策略

**核心原则：在用户获取信息的渠道中，选择 ROI 最高的 2-3 个集中投入。**

#### B2C 产品渠道矩阵

| 渠道类型 | 具体渠道 | 特点 | 适合阶段 |
|----------|----------|------|----------|
| **内容营销** | 公众号、小红书、B站、抖音 | 慢但长期 ROI 高 | 从 0 开始持续做 |
| **社群运营** | 微信群、Discord、知识星球 | 高粘性、高信任 | 早期种子用户 |
| **KOL/KOC** | 达人合作、测评 | 快但需要预算 | 发布期冲量 |
| **应用商店** | ASO、推荐位 | 自然流量稳定 | 持续优化 |
| **付费投放** | 信息流、搜索、社媒 | 快但烧钱 | PMF 后规模化 |
| **病毒传播** | 邀请机制、社交裂变 | 免费但需要产品支持 | 产品内嵌 |

#### B2B 产品渠道矩阵

| 渠道类型 | 具体渠道 | 特点 |
|----------|----------|------|
| **直销** | BD、电话销售 | 高客单价、长周期 |
| **内容营销** | 白皮书、案例研究、Webinar | 建立专业信任 |
| **SEO/SEM** | 搜索引擎优化/竞价 | 精准获客 |
| **合作伙伴** | 渠道商、集成市场 | 借力打力 |
| **PLG（产品驱动增长）** | 免费版自传播 | 低 CAC、高规模化 |

#### PLG（Product-Led Growth）模式

PLG 是近年 B2B SaaS 最热的 GTM 策略：

```
用户自助体验 → 个人喜爱 → 团队扩散 → 企业付费

代表产品：Slack、Notion、Figma、Zoom、Calendly
```

**PLG 的三个前提：**
1. 产品能自助上手（不需要实施团队）
2. 个人就能感受到价值（Aha Moment 短）
3. 有自然的团队协作传播路径（用一个人 → 团队都在用）

### 第五步：发布策略与节奏

#### 产品发布的三种节奏

| 策略 | 说明 | 适用场景 | 风险 |
|------|------|----------|------|
| **Big Bang（集中爆发）** | 集中所有资源在一天发布 | 有大量预算、需要造势 | 如果产品出 bug 全网翻车 |
| **Rolling Launch（滚动发布）** | 分阶段、分市场逐步发布 | 资源有限、需要验证 | 初期声量小 |
| **Soft Launch（软发布）** | 悄悄上线，积累数据后再推广 | 需要验证产品稳定性 | 缺乏话题性 |

#### 发布时间线（以一款 C 端 App 为例）

```
T-30 天：产品锁定 + GTM 方案定稿
  ├── 确定 beachhead 市场和核心渠道
  ├── 完成定价和包装设计
  └── 准备营销素材（视频、图文、KOL Brief）

T-21 天：内测 + 种子用户准备
  ├── 100-500 人封闭测试
  ├── 收集反馈、修复关键 bug
  └── KOL/KOC 内容预生产

T-14 天：预热阶段
  ├── 开放预注册 / waitlist
  ├── 社交媒体 teaser 释放
  ├── 媒体 exclusive 故事投放
  └── ASO 素材提交审核

T-7 天：渠道就位
  ├── KOL 内容排期确认
  ├── 投放计划和素材就绪
  ├── 客服话术培训
  └── 监控仪表盘搭建

T-0 发布日：
  ├── 全渠道推送（App Store / 各应用市场）
  ├── KOL 内容集中发布
  ├── 付费投放开启
  ├── 媒体稿发布
  └── 实时监控数据 + 用户反馈

T+1 ~ T+7：持续运营
  ├── 每日数据 review（下载、激活、留存、评分）
  ├── 快速修复问题版本
  ├── 收集用户好评（引导满意用户打分）
  └── 二次传播（用户故事、数据战报）

T+14 ~ T+30：复盘与迭代
  ├── GTM 复盘（哪个渠道效果最好/最差）
  ├── 调整渠道预算分配
  └── 规划下一波增长策略
```

### 第六步：指标体系与成功标准

**GTM 不是"上线了就成功了"，而是有明确的指标来衡量。**

| 阶段 | 核心指标 | 目标值示例 |
|------|----------|-----------|
| **发布首周** | 下载量/注册量 | 10,000+ |
| **首周激活率** | 注册到 Aha Moment 的比例 | > 40% |
| **D7 留存** | 7 天后仍活跃的用户比例 | > 30% |
| **D30 留存** | 30 天后仍活跃的用户比例 | > 15% |
| **App Store 评分** | 上线 2 周内 | > 4.2 |
| **CAC** | 综合获客成本 | < LTV/3 |
| **自然增长占比** | 非付费渠道用户比例 | > 30% |

## B2B vs B2C GTM 核心差异

| 维度 | B2C | B2B |
|------|-----|-----|
| **决策者** | 用户自己 | 多人决策（使用者+采购+IT） |
| **销售周期** | 秒级（即时下载） | 周-月级（Demo+POC+合同） |
| **客单价** | 低（0-100 元/月） | 高（数千-数万/月） |
| **核心渠道** | 应用商店、社媒、内容 | 销售、行业活动、转介绍 |
| **增长模式** | 广告 + 自然增长 | 销售 + PLG |
| **GTM 关键** | 激活率、留存率 | 销售效率、客户成功 |

## 实战案例：Notion 的 GTM 策略

### 背景

Notion 2016 年发布时，笔记市场竞争激烈（Evernote、OneNote、Google Docs）。它通过独特的 GTM 策略实现了突围。

### 关键策略

| 策略 | 执行 | 效果 |
|------|------|------|
| **Beachhead 选择** | 聚焦产品经理和设计师群体 | 这群人是工具传播的关键节点 |
| **PLG 模式** | 免费版功能强大，个人即可使用 | 个人喜爱 → 团队采购 |
| **模板生态** | 鼓励用户创建和分享模板 | UGC 内容自带传播 |
| **教育定价** | 学生和教育者免费使用 Pro 版 | 培养下一代用户习惯 |
| **社区运营** | 全球 Notion Community、Ambassador 计划 | 超级用户成为布道者 |
| **SEO 飞轮** | 模板页面大量被搜索引擎收录 | 持续获取自然流量 |

### 结果

- 2021 年估值 100 亿美元
- 月活用户从 2019 年 100 万增长到 2021 年 2000 万+
- 90% 的新用户来自口碑推荐和自然渠道

## 面试常见追问

### Q1：如果你的 GTM 策略上线后发现数据不达标，你怎么调整？

调整框架：

1. **定位问题环节**：是获客不够（流量问题）还是激活不行（产品问题）还是留存不行（价值问题）？
2. **渠道效率分析**：哪个渠道的 CAC 最高、留存最差？砍掉还是优化？
3. **用户反馈收集**：找 10-20 个新用户做访谈，了解他们为什么没留下来
4. **快速实验**：调整一个变量（定价、onboarding、渠道），2 周看数据
5. **决策点**：如果是 product 问题 → 回去改产品；如果是 market 问题 → 换 beachhead

### Q2：B2B 产品如何选择 PLG vs 销售驱动？

| 条件 | 偏 PLG | 偏销售驱动 |
|------|--------|-----------|
| 客单价 | < $500/月 | > $2000/月 |
| 部署复杂度 | 自助上手 | 需要实施/对接 |
| 决策链 | 个人或小团队 | 多部门审批 |
| 产品标准化 | 高度标准化 | 需要定制化 |
| 代表 | Slack、Figma、Calendly | Salesforce、Workday |

**很多产品走混合模式：PLG 获客 + 销售团队做大客户 upsell。**

### Q3：新产品发布时，应该先做 GTM 还是先做 PMF？

**先 PMF，后 GTM。** 但两者不是完全串行的：

```
PMF 阶段（50-200用户）
  └── 验证产品价值，同时初步了解目标用户画像

GTM 准备阶段（200-1000用户）
  └── 开始系统性规划 GTM，测试渠道

GTM 执行阶段（PMF 确认后）
  └── 全力推广，规模化获客
```

### Q4：GTM 策略中最大的坑是什么？

**最常见的五个坑：**

1. **目标用户太宽** — "所有人都能用"等于"没人会买"
2. **渠道太多太散** — 10 个渠道各做一点，不如 2 个渠道做透
3. **只看下载不看留存** — 发布首日 10 万下载，一周后留存 5%
4. **定价拍脑袋** — 不做竞品分析、不做价格敏感度测试
5. **发布即结束** — 上线只是开始，发布后的运营迭代才是关键

## 标准回答框架

```
1. 定义（30秒）：GTM 是什么，回答四个核心问题（卖给谁/怎么卖/卖什么/怎么推）
2. 六步框架（120秒）：市场定义 → 定位 → 定价 → 渠道 → 发布 → 指标
3. B2B vs B2C 差异（30秒）：决策链、销售周期、增长模式
4. PLG 模式（30秒）：概念 + 适用条件 + 案例
5. 案例（60秒）：用 Notion 或自己的项目说明完整 GTM 闭环
```

## 关键收获

1. **GTM 是全链路策略** — 不只是"怎么推广"，而是从市场选择到商业化的完整闭环
2. **Beachhead 策略是关键** — 先拿下一个小而精的市场，再扩张，而不是一上来就打大盘
3. **渠道选择做减法不做加法** — 找到 ROI 最高的 2-3 个渠道集中投入
4. **PLG 是 SaaS 的最佳 GTM** — 让产品自己卖自己，但需要满足三个前提条件
5. **发布节奏要有策略** — 不是所有产品都适合 Big Bang，根据资源和产品成熟度选择
6. **GTM 没有标准答案** — 不同产品、不同市场、不同阶段需要不同的 GTM 策略
