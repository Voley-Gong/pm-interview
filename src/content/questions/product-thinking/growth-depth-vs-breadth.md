# 产品增长应该做深核心场景还是拓展新场景？

## 面试场景

> 面试官：你的产品已经验证了PMF，核心功能有一定用户基础。现在面临一个战略选择：是把资源用来打磨核心场景、提升留存，还是拓展新的使用场景、获取更多用户？你怎么思考这个决策？

这是一道Level 4的高阶产品思维题，考察的不是"正确答案"（因为没有标准答案），而是PM的**系统思考能力、战略判断框架、以及对增长底层逻辑的理解深度**。

---

## 核心概念解析

### "做深"与"做宽"的本质

| 维度 | 做深（Depth） | 做宽（Breadth） |
|------|--------------|----------------|
| 增长杠杆 | 提升LTV、增加留存 | 扩大TAM、获取新用户 |
| 核心假设 | "用户需要更多价值" | "产品可以服务更多场景" |
| 主要风险 | 边际效用递减、过度工程 | 焦点涣散、资源稀释 |
| 成功指标 | 留存率、使用深度、NPS | 用户量、场景渗透率、获客效率 |
| 适用阶段 | PMF后早期、防御期 | 增长期、核心场景成熟后 |
| 组织要求 | 深度理解核心用户 | 跨场景的用户研究能力 |

### 为什么这是产品思维的高阶题？

因为这个问题**没有通用最优解**。正确的答案取决于：
- 产品所处生命周期阶段
- 竞争格局
- 团队能力边界
- 核心指标的当前表现

面试官想看你如何在**信息不完整的情况下，构建决策框架并做出有逻辑的判断**。

## 决策框架：做深做宽的 STRIDE 模型

### S - Saturation Assessment（饱和度评估）

第一步：评估核心场景的**剩余增长空间**。

```
核心场景饱和度检查清单：

□ 核心场景的渗透率（DAU/MAU中使用了核心功能的占比）
□ 核心场景的使用频次趋势（仍在增长？还是趋于平缓？）
□ 用户满意度（NPS、功能满意度评分）
□ 核心场景的留存曲线（是否稳定？仍有流失？）
□ 竞品在核心场景上的差距（领先多少？追赶速度？）

解读：
- 渗透率 < 60% + 满意度不高 → 做深（核心场景还有大空间）
- 渗透率 > 80% + 满意度高 → 考虑做宽（核心场景接近天花板）
- 留存曲线仍有下滑 → 做深（留存问题没解决，做宽只是漏更多的水）
```

### T - TAM Expansion Analysis（市场扩展分析）

评估做宽的市场潜力：

```
新场景评估框架：

1. 新场景与核心场景的协同度
   - 高协同：新场景是核心场景的自然延伸（如：微信从聊天→朋友圈→支付）
   - 低协同：新场景是完全不同的使用case（如：音乐App做社交）

2. 新场景的市场规模
   - 新场景能带来多少增量用户？
   - 这些用户的LTV与核心用户相比如何？

3. 新场景的进入壁垒
   - 需要多少额外开发量？
   - 需要什么新能力（运营、BD、内容）？
   - 竞争对手在这个场景中的位置？
```

### R - Risk Asymmetry（风险不对称性）

做深和做宽的风险是不对称的：

```
做深的风险：
├── 过度优化——投入大量资源但边际收益递减
├── 近视——专注于现有用户，错过市场趋势变化
└── 竞品侧翼攻击——对手通过新场景获取新用户，最终包抄核心

做宽的风险：
├── 资源稀释——核心场景失去投入，被竞品追上
├── 认知混乱——用户不清楚产品到底是做什么的
├── 组织复杂度——团队需要同时维护多个场景
└── 最致命：核心未稳就扩张，两个方向都做不好

原则：做宽的风险通常大于做深。因为做深的失败是"不够好"，
      做宽的失败是"什么都做了但什么都不行"。
```

### I - Indicator Diagnosis（指标诊断）

用数据辅助决策：

| 指标表现 | 信号 | 建议方向 |
|---------|------|---------|
| 高渗透 + 高留存 + 低新用户增长 | 核心场景成熟 | 做宽探索新增长曲线 |
| 中渗透 + 留存有提升空间 | 核心场景未挖透 | 做深，提升留存和深度 |
| 低渗透 + 低留存 | 核心场景本身有问题 | **必须做深**，做宽等于自杀 |
| 高渗透 + 低留存 | 价值主张有天花板 | 做宽找新价值点 |
| 新场景早期数据好 | 用户有真实需求 | 值得做宽投入验证 |

### D - Decision Sequencing（决策排序）

关键洞察：**做深和做宽不是同时选择的，而是有先后顺序的（Sequencing）**。

```
经典Sequencing模式：

阶段1（PMF → 增长初期）：
  ──── 做深核心 ────→
  目标：验证核心价值、提升留存、建立口碑

阶段2（增长期）：
  ──── 做深（70%）+ 做宽探索（30%）──→
  目标：在核心稳固的前提下，探索第二增长曲线

阶段3（成熟期）：
  ──── 做宽（多场景/多产品线）──→
  目标：扩大TAM，防御竞争

阶段4（防御/转型期）：
  ──── 重新做深（新核心）──→
  目标：在新领域建立深度壁垒
```

**Amazon的经典案例**：
```
做深：图书（1995-1998）→ 把在线购书做到极致
做宽：全品类电商（1999-2005）→ 从图书扩展到所有品类
做深：Prime会员体系（2005-2010）→ 深度绑定用户
做宽：AWS / Kindle / Alexa（2006-至今）→ 完全新场景
```

### E - Execution Evaluation（执行可行性评估）

最后一步：即使方向正确，执行能力是否到位？

**做深的执行要求**：
- 深度用户研究能力（能发现用户自己都说不清的痛点）
- 精细化设计能力（微优化、A/B测试体系）
- 长期耐心（深度的回报是渐进的）

**做宽的执行要求**：
- 快速验证能力（MVP、POC、用户测试）
- 资源调配能力（在不影响核心的前提下分出资源）
- 强大的项目管理（多线并行）

## 实战案例

### 案例分析：网易云音乐的选择

**背景时间点**：2014-2015年，网易云音乐已有一定用户基础，核心场景是"发现音乐"（歌单+推荐）。

**面临的决策**：
- 做深：继续优化推荐算法、歌单生态、评论区氛围
- 做宽：扩展直播、社交、短视频、播客等新场景

**实际选择和分析**：

```
Phase 1 - 做深（2014-2017）：
  深耕"音乐发现+情感共鸣"核心场景
  ├── 推荐算法持续优化
  ├── 评论区文化精心培育
  ├── UGC歌单生态建设
  └── 结果：差异化定位成功，用户深度极高

Phase 2 - 做宽尝试（2018-2020）：
  在核心稳固后尝试扩展
  ├── 直播（属于音乐场景延伸，协同高）
  ├── 播客（音频场景延伸，中协同）
  └── 社交功能（社区已有基础，协同较高）

Phase 3 - 回归做深+防御（2021-至今）：
  面对抖音音乐的竞争
  ├── 做深：版权竞争力、音乐人生态
  └── 做宽：AI DJ、空间音频等体验创新
```

**教训**：
- 在核心场景（音乐发现）尚未绝对领先时就做宽（直播），分散了资源
- 竞品（抖音）通过更宽的场景（短视频+音乐）侧翼包抄
- 如果早期更专注于做深推荐和音乐人生态，竞争壁垒可能更高

## 面试常见追问

### 追问1："如果CEO坚持要做宽，但你认为应该做深，怎么argue？"

**回答思路**：
1. 不要直接反对——先理解CEO看到的机会（他可能看到了你没看到的市场信号）
2. 用数据说话：
   - "目前核心场景的留存率是35%，行业benchmark是50%。如果我们现在做宽，新进来的用户留存可能更低，因为核心体验还没到位。"
3. 提出验证方案：
   - "我们可以用最小投入做一个新场景的POC——花2周做一个landing page测试用户兴趣，如果转化率超过8%，我全力支持做宽。"
4. 如果CEO仍然坚持：转为执行者角色，但确保设置review checkpoint

### 追问2："什么时候做宽是对的？"

**回答思路**：
四个条件同时满足时，做宽是合理的：
1. **核心场景接近饱和**：渗透率>70%，留存稳定，竞品难以追赶
2. **新场景有真实需求**：不是PM拍脑袋，而是有用户数据支撑
3. **新场景与核心有协同**：能复用核心能力或用户基础
4. **组织能力到位**：团队能支撑多线作战，不会导致核心失守

经典案例：微信从即时通讯（核心）→ 朋友圈（做深社交）→ 公众号（做宽内容）→ 支付（做宽金融）→ 小程序（做宽平台）。每一步做宽都是在核心场景极其稳固后才进行的。

### 追问3："你怎么避免做深时的过度工程？"

**回答思路**：
1. 做深≠功能堆砌。做深的核心是**提升核心场景的成功率和体验**，不是加更多功能
2. 用投入产出比判断：
   - 每一轮优化预估带来的指标提升 vs 投入的工程量
   - 当ROI低于做宽的预期ROI时，就是转向做宽的信号
3. 设置检查点：每季度评估核心场景的优化是否还在产生有意义的提升

## 标准回答框架

```
1. 重构问题（30秒）
   "做深还是做宽不是二选一的选择题，
   而是一个 sequencing（排序）问题。
   关键是在正确的阶段做正确的选择。"

2. 展示框架（60秒）
   "我会用STRIDE模型做决策：
   - Saturation：评估核心场景饱和度
   - TAM：分析新场景的市场潜力
   - Risk：评估做深/做宽的风险不对称性
   - Indicator：用核心指标辅助判断
   - Decision Sequencing：确定先后顺序
   - Execution：评估执行可行性"

3. 给出判断（60秒）
   "在[当前产品阶段]下，我倾向于[做深/做宽]，
   因为[核心论据]。但这不是永久的，
   当[触发条件]出现时，我会转向另一个方向。"

4. 验证方案（30秒）
   "为了降低决策风险，我建议先做一个低成本的验证：
   [具体的MVP/POC方案]。"

5. 长期思考（30秒）
   "无论做深还是做宽，最关键的是保持对用户价值的聚焦。
   做深是为了让核心用户更依赖你，
   做宽是为了让更多用户从你这里获得价值。
   两者殊途同归。"
```

## 进阶认知

### 贝索斯的"两种增长"理论

```
Type 1 增长（做深型）：
  深化现有价值 → 提升用户依赖度 → 高质量增长
  特点：慢但扎实，复利效应

Type 2 增长（做宽型）：
  拓展新场景 → 增加用户基数 → 规模化增长
  特点：快但需要核心支撑，否则是泡沫

贝索斯原则：永远优先Type 1增长，
            因为Type 1会自然带来Type 2，
            但反过来不成立。
```

### 互联网产品的"深度护城河"清单

做深能建立的壁垒：
- **数据壁垒**：使用数据越多→推荐越准→体验越好→更多使用（飞轮效应）
- **习惯壁垒**：用户形成深度使用习惯→切换成本极高
- **生态壁垒**：UGC内容/社区文化→竞争对手无法复制
- **网络效应**：核心场景做深后，用户间的网络效应更强

这些壁垒都是做深才能建立的。做宽可以扩大规模，但很少能建立深度壁垒。

## 关键收获

1. **做深做宽是sequencing问题，不是either/or**：正确的顺序是先深后宽、在核心稳固后再扩展
2. **STRIDE模型**：系统化的决策框架——先评估饱和度，再看市场潜力，评估风险不对称性，用指标辅助，做排序决策
3. **风险不对称性**：做宽的失败（平庸化）比做深的失败（不够好）更致命
4. **验证驱动决策**：无论选择哪个方向，都用低成本验证降低不确定性
5. **面试核心信号**：面试官想看到三个层次的思考——
   - Level 1：能说出做深和做宽的利弊
   - Level 2：能用框架做结构化分析
   - Level 3（目标）：能结合产品阶段、竞争格局、组织能力做动态判断
6. **最高境界**：理解"做深是因，做宽是果"——真正伟大的产品增长，都是因为在核心场景上做到了极致，才自然扩展到更宽的领域
